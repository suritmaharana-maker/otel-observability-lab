Concept Diagram — The Observability Bridge

The thesis: App observability tells you something is broken. Network observability tells you why. The OTel pipeline is the bridge. Phase 5 adds the LLM that reads both and explains it in plain English.

The three-layer stack

Phase 5 — the AIOps loop

How /diagnose works end to end
1. Fault injected — CiliumNetworkPolicy blocks gateway→product-svc:8001
2. Signals flow — hubble_drop_total spikes in Dash0, OTel SDK records 502 errors
3. GET /diagnose — llm-svc queries Dash0 Prometheus API for 4 signals
4. Bedrock called — signals + SRE context prompt → Nova Micro
5. RCA returned in 2.2s — root cause, confidence, evidence, kubectl command
6. Bridge call avoided — what used to take 45 minutes now takes 2 seconds

Beyla TCP retransmit — verified finding

Gemini hallucinated — verified June 17, 2026 against Grafana docs

Beyla 3.20.0 exposes only beyla_network_flow_bytes at its Prometheus endpoint. The tcp_retransmit_skb tracepoint is used internally for L7 span correlation but is NOT a user-facing metric. TCP retransmit visibility requires a custom eBPF program — a Phase 6 contribution opportunity.
