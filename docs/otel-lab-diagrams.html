Concept Diagram — The Observability Bridge

The gap between APM and network observability is the root cause of long bridge calls. This architecture closes it.

The Three Layers

Layer 1 — Application
OTel SDK + OpenLLMetry
Traces · Logs · gen_ai.* spans · Cost per request
Layer 2 — Network
Cilium Hubble + Beyla
Flow bytes · POLICY_DENY · L7 spans · TCP timing
Layer 3 — Infrastructure
hostmetrics + OTel Collector
CPU · Memory · Disk · One pipeline for all signals

The Blindspot — Phase 3 Proof

5,016ms timeout · 502 error
No product-svc span · No SQL span · No root cause
eBPF sees (root cause)
TCP POLICY_DENY · port 8001
gateway → product-svc blocked · hubble_drop_total spike

Beyla TCP Retransmit — Verified Finding

Gemini hallucinated — verified June 17, 2026

Beyla 3.20.0 exposes only beyla_network_flow_bytes_total at its Prometheus endpoint. The tcp_retransmit_skb tracepoint is used internally for L7 correlation but NOT exposed as a user metric. TCP retransmit visibility requires a custom eBPF program — a Phase 6 opportunity.
